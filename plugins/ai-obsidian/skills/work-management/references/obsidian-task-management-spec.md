# Obsidian 任务管理规范

## 任务库位置

- 默认根路径：当前项目根目录。
- 用户显式指定路径时，以用户指定路径为准。
- 任务根目录：`<项目根目录>/任务`
- 年度目录：`<项目根目录>/任务/{YYYY}`
- 计划文件：`<项目根目录>/任务/{YYYY}/计划中.md`
- 归档文件：`<项目根目录>/任务/{YYYY}/已归档.md`
- 年份按任务开始日期 `🛫` 所在年份确定；查询跨年范围时读取多个年度目录。

## 文件组织

每个任务文件按项目分区：

```markdown
# 2026 计划中任务

## 项目A

- [ ] 父任务 #project/项目A 🛫 2026-06-22 📅 2026-06-30
	- 父任务说明
	- [ ] 子任务一 #project/项目A 🛫 2026-06-22 📅 2026-06-24
	- [/] 子任务二 #project/项目A 🛫 2026-06-24 📅 2026-06-27
		- 子任务说明

## 项目B

- [?] 等待确认的任务 #project/项目B 🛫 2026-06-20 📅 2026-06-25
	- 等待接口确认
```

- 缺少年度目录或文件时，可以创建。
- 创建文件时保留一级标题：`# {YYYY} 计划中任务` 或 `# {YYYY} 已归档任务`。
- 项目分区不存在时，在文件末尾追加 `## 项目名`。
- 同一项目内保留父子层级；不要把子任务移动成顶层任务。

## 项目集合（运行时动态发现，不写死）

项目分区不是预置白名单，而是从当前任务库实时发现：扫描 `<项目根目录>/任务/{YYYY}/计划中.md`、`已归档.md` 中已存在的 `## 项目名` 分区，得到当前允许的项目集合。

- 新任务的项目命中已有分区时，直接归入。
- 用户给出的项目名不在已有集合内时，先确认再写入：是要**新建分区**，还是某个已有项目的**别名/笔误/历史叫法**（大小写、简称等）需归一化到已有名——以此避免产生重复分区。
- 不要自行臆造或静默新增项目分区；也不要预设任何具体项目名或固定映射。

## 任务行格式

```markdown
- [状态] 任务标题 #project/项目名 优先级 🛫 YYYY-MM-DD 📅 YYYY-MM-DD ✅ YYYY-MM-DD
	- 可选说明
```

- 必填：任务标题、项目、开始日期 `🛫`、计划完成日期 `📅`。
- 可选：优先级、完成日期 `✅`、说明/备注。
- checkbox 任务行只放任务标题、项目标签、优先级、循环标记和日期等 Obsidian Tasks 可查询字段。
- 说明/备注必须写在任务下方一层缩进的普通列表行，例如 `<Tab>- 等待接口确认`；不要追加到 checkbox 任务行尾，也不强制使用 `备注：` 前缀。
- 补充说明也使用任务下方一层缩进的普通列表行，不带 checkbox。
- 项目标签使用 `#project/项目名`，与所在 `## 项目名` 分区保持一致。

## 状态标记

| 标记 | 含义 |
| --- | --- |
| `[ ]` | 未开始 / 未完成 |
| `[/]` | 进行中 |
| `[?]` | 阻塞 / 待确认 |
| `[x]` | 已完成 |

- 新建任务默认使用 `[ ]`。
- 完成任务时改为 `[x]`，并补 `✅ YYYY-MM-DD`。
- 完成任务不自动归档。
- 完成子任务后，如果同一父任务下的 checkbox 子任务全部完成，可以自动同步完成父任务，父任务完成日期取子任务中最晚的 `✅` 日期。
- 循环任务、长期支持任务或持续进行类父任务不自动同步完成，除非用户明确要求。
- 手动完成父任务但存在未完成子任务时，必须先提醒用户并确认。

## 子任务规则

- 每一层任务层级使用 1 个 Tab 缩进，不使用空格缩进。
- 父任务和子任务都使用 checkbox，便于 Obsidian Tasks 查询。
- checkbox 行表示任务；普通列表行表示说明、备注或补充记录。
- 说明/备注属于哪一级任务，就缩进到该任务下方一层：顶级任务说明缩进 1 个 Tab，子任务说明缩进 2 个 Tab。
- 子任务默认继承父任务项目，但写入时仍显式保留 `#project/项目名`。
- 完成子任务后可以按状态标记规则自动判断父任务是否同步完成。
- 调整父任务项目时，默认同步调整子任务项目；用户明确要求只改父任务时除外。

## 归档规则

- 归档必须由用户明确触发，例如“归档这个任务”“归档项目A已完成任务”“归档上周完成任务”。
- 归档时从 `计划中.md` 移动到同年度 `已归档.md`。
- 恢复时从 `已归档.md` 移回 `计划中.md`，并按用户要求设置状态。
- 父任务已完成时，归档默认移动整个任务树。
- 父任务未完成但部分子任务已完成时，执行部分归档：`计划中.md` 保留父任务和未完成子任务，`已归档.md` 写入同一个父任务作为上下文容器，并只放已完成子任务。
- 部分归档中的父任务保留真实状态，不补 `✅`，避免误判父任务整体完成。
- 恢复父任务或部分子任务前必须确认恢复范围。

## 查找和匹配

- 先按用户给出的项目、标题、父任务、日期范围缩小候选。
- 标题不唯一时，不直接修改；要求用户补充项目、日期或父任务。
- 查看任务时默认读取当前年度 `计划中.md`。
- 用户要求历史、已完成、归档或跨年范围时，同时读取相关年度的 `已归档.md`。
- 日期范围筛选要同时考虑 `🛫`、`📅`、`✅`，并保留父子层级。
