# 任务调整流程（动词层）

调整已存在任务的**动作序列**——先定位、再动手。记录长什么样（状态标记、任务行、父任务进度条、项目集合、归档规则）见 `task-format.md`，本文件不复述格式，只写"先做什么再做什么"。

所有调整操作都先走**定位协议**（第 1–3 步），再进各自骨架（A/B/C）分叉。

---

## 定位协议（所有调整的前置闸）

动手改盘前必须锁死目标行：**项目缩范围 → 命中数分流 → 原行回显确认**。

### 第 1 步：按"项目 + 标题"缩小候选

调整口令至少带**项目**——任务库按 `## 项目名` 分区，项目一给就把搜索范围锁进一个分区。

- 口令形如：`完成：{项目} / {标题}`。
- 只给标题不给项目：全库 Grep 标题，命中多个直接进第 3 步消歧，不猜。

### 第 2 步：按命中数分流

| 命中数 | 处理 |
| --- | --- |
| 恰好 1 条 | 进第 3 步回显确认 |
| 0 条 | 报"未找到"，列出该项目分区下相近标题，不臆造 |
| ≥2 条 | 不猜，用宿主原生交互工具让用户补 `日期 / 父任务 / 优先级` 或直接选行 |

### 第 3 步：回显整行 + 拟改动，确认后才写

动手前把**定位到的原行**和**改完的新行**并排回显，让用户看到的是真实读到的那一行，不是脑补的：

```
定位到（订单系统）：
  - [/] 接入支付回调 🔼 🛫 2026-07-17 📅 2026-07-25
将改为：
  - [x] 接入支付回调 🔼 🛫 2026-07-17 📅 2026-07-25 ✅ 2026-07-19
```

行不对，立刻停。三关过了才写盘。

---

## 骨架 A：改状态

定位口令：`完成：{项目} / {标题}`、`阻塞：{项目} / {标题}` 等。

走完定位协议第 1–3 步后：

1. 目标状态 = `[ ]` / `[/]` / `[?]` / `[x]`（含义见 task-format「状态标记」）。
2. 目标为 `[x]`：行尾补 `✅ {今天}`；改成其余状态不动 `✅`。
3. 连带判断：
   - 该行是**子任务**，且同父下 checkbox 子任务这次全部 `[x]` → 提示"父任务可同步完成，`✅` 取最晚子任务日期"，确认再动父行（循环 / 长期类父任务不提示，见 task-format）。
   - 该行是**父任务**，改 `[x]` 但仍有未完成子任务 → 先提醒有 N 个未完成子任务，确认才写。
4. 该行有子任务 → 按 task-format「父任务进度条」重算父进度条（重算规则见骨架 C 第 5 步）。

---

## 骨架 B：移项目

定位口令：`移动：{源项目} / {标题} → {目标项目}`。

走完定位协议第 1–3 步后：

1. 校验目标 `## {目标项目}` 分区（按 task-format「项目集合」扫描现有分区）：
   - 命中现有分区 → 直接作落点。
   - 不在集合内 → 先确认"新建分区"还是"已有项目别名"（归一化到已有名，不造重复分区）。
2. 整棵移动：父任务 + 其下全部子任务 + 各自 blockquote 说明，一起从源分区剪切、追加到目标分区末尾；层级缩进原样保留（1 Tab/层，子任务不许被拉平成顶层）。
3. 进度条随任务原样带走（子任务集合没变，不重算）。
4. 回显源分区剩余 / 目标分区新增，两侧都亮出来确认。

---

## 骨架 C：改结构（增删 / 拆分 / 合并子任务）

定位口令：`拆分：{项目} / {父标题}`、`{父标题} 下加子任务 / 删子任务` 等。

走完定位协议第 1–3 步后：

1. 增子任务：在父行下按缩进 +1 Tab 追加 `- [ ]`，默认 `[ ]`、必填 `🛫` `📅`。
2. 删子任务：删该 checkbox 行 + 它下方一层的 blockquote 说明。
3. 叶子任务首次加子任务 → 升级为父任务，标题后新增 `<span class="tp">…</span>`。
4. 删到只剩 0 个子任务 → 父任务降级为叶子，移除整段进度条。
5. **【强制】重算父进度条**（细节见 task-format「父任务进度条」）：
   - 每子任务一段 `<i>`，按 `已完成 → 进行中 → 阻塞 → 未开始` 排序。
   - `[x]`→`s-done`、`[/]`→`s-doing`、`[?]`→`s-blk`、`[ ]`→`s-todo`。
   - 已完成 / 进行中 / 阻塞用 `▰`，未开始用 `▱`。
6. 回显父任务整块（父行 + 全部子任务）确认。

---

## 三骨架的防偏差重心

| 骨架 | 共享前置 | 定位后重心 |
| --- | --- | --- |
| A 改状态 | 定位协议 1–3 | 连带父任务（子任务全完成 / 父任务有未完成子任务） |
| B 移项目 | 定位协议 1–3 | 目标分区校验 + 整棵不散架 |
| C 改结构 | 定位协议 1–3 | 增删后强制重算进度条 |
