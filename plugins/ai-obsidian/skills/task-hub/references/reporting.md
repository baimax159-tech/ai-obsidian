# Obsidian 报告生成规范

从任务记录生成工作报告。报告与任务放在同一年度目录，每次生成插入到对应报告文件头部，最新在最上面。

## 数据模型

- 任务文件：`任务/{YYYY}/计划中.md`；归档文件：`任务/{YYYY}/已归档.md`。
- 任务按 `## 项目名` 分区，任务行是 checkbox；父任务为顶层 checkbox，子任务每层 1 Tab 缩进。
- 父任务标题后可能带进度条 `<span class="tp">…</span>`（纯展示）——读取标题、日期、统计时一律忽略它，不并进标题、不计入结果。

## 报告文件

| 报告 | 文件 |
| --- | --- |
| 日报 | `任务/{YYYY}/日报.md` |
| 周报 | `任务/{YYYY}/周报.md` |
| 月报 | `任务/{YYYY}/月报.md` |
| 季度报告 | `任务/{YYYY}/季度报告.md` |
| 年度报告-半年 | `任务/{YYYY}/半年报.md` |
| 年度报告-全年 | `任务/{YYYY}/年度报告.md` |

写入规则：

- 文件不存在时创建并写总标题：`# {YYYY} 日报`、`# {YYYY} 周报` 等。
- 文件存在时保留第一行总标题，新报告插入到总标题下面，最新在最上面。
- 同一周期已有报告时，默认仍在头部新增一条；只有用户明确要求覆盖时才替换对应条目。

## 日期范围

| 报告 | 范围 |
| --- | --- |
| 日报 | 指定日期，默认今天 |
| 周报 | 指定日期所在工作周，周一到周五 |
| 月报 | 指定月份 |
| 季度报告 | `YYYY-Q1`…`YYYY-Q4` |
| 年度报告-半年 | `YYYY-H1` 或 `YYYY-H2` |
| 年度报告-全年 | 指定年份 |

筛选规则：

- 日期范围跨多年时读取多个年度目录。
- 日报只纳入 `🛫 <= 报告日期 <= 📅` 的任务。
- 周报及以上纳入范围内开始、计划完成、实际完成或持续中的任务。
- 任务已归档但 `✅` 或任务周期落在报告范围内时必须纳入。

## 状态映射

| 标记 | 报告状态 |
| --- | --- |
| `[x]` | 已完成 |
| `[/]` | 进行中 |
| `[?]` | 阻塞 |
| `[ ]` | 未开始 |

- 完成日期来自 `✅`，计划完成来自 `📅`，开始来自 `🛫`。
- 备注来自任务下方缩进 blockquote（`> ...`）；兼容读取历史普通列表行注释与行尾 `备注：...`。

## 日报格式

精简、直观、明确。只按子任务逐条列出，不显示日期，不做表格，不写长段总结。

```markdown
## 2026-06-22 日报

【数据任务】基础依赖解析组件开发：当前使用 a 技术解决，目前进行中
【系统管理】报告技能重构：已确认日报与周报格式
```

- 项目按项目名排序。每条优先合并同项目、同父任务或同类工作的子任务；父任务无子任务时把父任务当一条。
- 格式固定：`【项目名】子任务标题：描述`。描述优先来自备注，没有则用状态和上下文生成一句简短说明。
- 描述是一句短句，单条 20-35 汉字左右，日报一般 3-5 条。不显示日期，不输出未落在当天周期内的任务。

## 周报格式

正式格式使用纯 HTML `<table>` 三列表格：`项目 | 任务 | 完成情况`。项目列用 `rowspan` 跨行合并——一个项目有几个顶级任务，`rowspan` 就等于几，项目名只在该项目首个 `<tr>` 出现一次。零插件依赖，Obsidian 阅读视图 / GitHub / 任意 Markdown 环境渲染一致。

```markdown
## 2026-06-22 ~ 2026-06-26 周报

<table>
<thead>
<tr><th>项目</th><th>任务</th><th>完成情况</th></tr>
</thead>
<tbody>
<tr>
  <td rowspan="2">项目A</td>
  <td><strong>【进行中】 顶级任务一</strong>
    <ul>
      <li>任务组【完成日期：YYYY-MM-DD】<ul><li>已完成子任务说明。</li></ul></li>
      <li>子任务组【进行中：30%】<ul><li>当前进展说明。</li></ul></li>
    </ul>
  </td>
  <td>
    <ul>
      <li>本周完成：概括完成内容。</li>
      <li>本周进行中：概括当前情况。</li>
      <li>后续安排：概括下周方向。</li>
    </ul>
  </td>
</tr>
<tr>
  <td><strong>【已完成】 顶级任务二</strong>
    <ul><li>任务组【完成日期：YYYY-MM-DD】<ul><li>交付说明。</li></ul></li></ul>
  </td>
  <td>
    <ul>
      <li>本周完成：概括完成内容。</li>
      <li>后续安排：概括下周方向。</li>
    </ul>
  </td>
</tr>
</tbody>
</table>
```

生成规则：

- 标题用工作周范围：`## YYYY-MM-DD ~ YYYY-MM-DD 周报`（通常周一到周五）；标题下不再单独写时间范围行。
- 日报和周报默认只读 `计划中.md`，不扫 `已归档.md`，避免部分归档父任务重复统计。
- 不统计 `[ ]` 未开始任务。扫描范围同时考虑完成时间和规划时间：`✅` 落在本周，或计划区间与本周有交集（`🛫 <= 周五` 且 `📅 >= 周一`）即纳入。
- 行粒度是项目下的顶级任务，不要把子任务拆成独立表格行。顶级任务命中条件：自身命中，或任一子任务命中。
- 表头用 `<table><thead>` 写死 `项目 | 任务 | 完成情况`；项目列用 `<td rowspan="N">` 合并，N = 该项目本周纳入的顶级任务数，项目名只在首个 `<tr>` 出现，其余顶级任务的 `<tr>` 不再写项目 `<td>`。
- 任务标题格式 `**【状态】 顶级任务标题**`，状态用中文 `【进行中】`/`【已完成】`/`【阻塞】`。
- 任务列写具体事实：做了哪些、进度、完成日期（`【完成日期：YYYY-MM-DD】`）、进行中进度（`【进行中：NN%】` 或 `【进行中：计划YYYY-MM-DD】`）。百分比只能来自任务记录或用户明确给出，不推算。
- 任务列 / 完成情况列内部用 HTML `<ul><li>...</li></ul>` 保证渲染稳定，子任务多时拆多条。
- 完成情况列写父任务本周总体概括，默认三条：`本周完成：…`、`本周进行中：…`、`后续安排：…`；不逐条复制任务列。
- 明细只能来自任务标题、说明、子任务、状态和日期关系；不编造完成结果、比例、指标或依赖方。
- 同项目内按父任务层级、子任务日期和状态排序：已完成、进行中、阻塞。

### 周报测试规则

- 测试阶段先用增加模式生成候选周报，不覆盖已有正式段落。用户确认后才替换正式周报或删旧候选。

## 月报格式

```markdown
## 2026-06 月报

### 项目A

- 主要完成：
  - 任务一：基于任务记录归纳成果
- 持续推进：
  - 任务二：说明当前进展
- 问题与风险：
  - 任务三：说明阻塞或待确认事项
- 下月关注：
  - 从未完成、进行中和后续计划任务中归纳
```

## 季度报告格式

只用于 `Q1/Q2/Q3/Q4`，写入 `任务/{YYYY}/季度报告.md`。材料化结构，不用表格，按项目分段。

```markdown
## 2026-Q2 季度报告

### 项目A

- 围绕业务贡献总结季度完成内容，把可衡量指标融入描述（完成任务数、验证规模、接口量、问题修复数）。
- 说明仍在推进或需下季度关注的事项。

### 下季度规划

- 按方向写目标、重点工作、预期贡献和建议跟踪指标。
```

- 标题 `## YYYY-QN 季度报告`；周期只能是 Q1-Q4。必须按项目归纳，不逐条罗列。
- 指标必须来自任务记录或用户提供，融入描述，不单独列"指标清单"。不编造比例、收益、数量。

## 年度报告格式

半年报告写 `任务/{YYYY}/半年报.md`，全年报告写 `任务/{YYYY}/年度报告.md`。按项目归纳，材料化要点风格，不写流水账。

### 半年报告

```markdown
## 2026-H1 半年报告

### 一、项目A

- **核心方向：**
  - 材料风格总结重点工作，融合可参考指标。
- **业务贡献：**
  - 说明对业务稳定 / 迁移效率 / 数据质量 / 平台能力的支撑。
- **不足与后续：**
  - 说明未闭环事项和下半年改进方向。

### 亮点、不足以及改进计划

- **亮点：** …
- **不足：** …
- **改进计划：** …
```

- 标题 `## YYYY-H1 半年报告` 或 `H2`。按项目分段，"编号标题 + 粗体小项 + 嵌套要点"。用户要求字数时严格按上下限。指标融入正文。

### 全年报告

```markdown
## 2026 全年报告

### 项目A

- 年度成果：基于已完成任务归纳
- 建设与支撑：基于父子任务层级归纳
- 问题与风险：基于阻塞和待确认任务归纳

### 下一年度计划

- 从未完成任务和后续计划中整理
```

- 标题 `## YYYY 全年报告`。按项目归纳全年成果、建设支撑、问题风险和下一年度计划。用户只给 `H1/H2` 时应生成半年报告。

## 质量要求

- 日报短、直观；周报具体、可读；月报及以上按项目归纳，不堆砌全部任务。
- 不编造任务、完成日期、结果、指标或进度百分比。任务记录没有量化指标就不写。
- 任务描述缺失时，允许用父任务、项目和状态补足上下文，但不改变事实。
