# Obsidian 任务格式契约

task-hub 的创建 / 修改 / 归档 / 报告都读写同一份记录，格式必须逐字一致，否则跨操作查找、筛选、统计会失配。

## 任务库位置

- 默认根路径：当前项目根目录。用户显式指定路径时以用户指定为准。
- 任务根目录：`<项目根目录>/任务`
- 年度目录：`<项目根目录>/任务/{YYYY}`
- 计划文件：`<项目根目录>/任务/{YYYY}/计划中.md`
- 归档文件：`<项目根目录>/任务/{YYYY}/已归档.md`
- 年份按任务开始日期 `🛫` 所在年份确定；跨年查询读取多个年度目录。

## 文件组织

每个任务文件按项目分区：

```markdown
# 2026 计划中任务

## 项目A

- [ ] 父任务 <span class="tp"><i class="s-doing">▰</i><i class="s-todo">▱</i></span> 🛫 2026-06-22 📅 2026-06-30
	> 父任务说明
	- [ ] 子任务一 🛫 2026-06-22 📅 2026-06-24
	- [/] 子任务二 🛫 2026-06-24 📅 2026-06-27
		> 子任务说明

## 项目B

- [?] 等待确认的任务 🛫 2026-06-20 📅 2026-06-25
	> 等待接口确认
```

- 缺少年度目录或文件时可以创建。创建时保留一级标题：`# {YYYY} 计划中任务` 或 `# {YYYY} 已归档任务`。
- 项目分区不存在时，在文件末尾追加 `## 项目名`。
- 同一项目内保留父子层级；不要把子任务移动成顶层任务。

## 项目集合（操作前先扫描，不写死）

项目分区不是预置白名单。每次涉及项目的操作（新增任务、移动/调整项目、按项目筛选、生成报告）前，先扫描当前任务库 `任务/{YYYY}/计划中.md`、`已归档.md` 中已存在的 `## 项目名` 分区，得到当前允许的项目集合。

- 命中已有分区时直接归入。
- 用户给出的项目名不在已有集合内时，先确认再写入：是要**新建分区**，还是某个已有项目的**别名 / 笔误 / 历史叫法**（大小写、简称等）需归一化到已有名——避免产生重复分区。
- 不要自行臆造或静默新增项目分区；也不要预设任何具体项目名或固定映射。

## 任务行格式

```markdown
- [状态] 任务标题 优先级 🛫 YYYY-MM-DD 📅 YYYY-MM-DD ✅ YYYY-MM-DD
	> 可选说明
```

- 必填：任务标题、项目（靠所在 `## 项目名` 分区归属，任务行不带项目标签）、开始日期 `🛫`、计划完成日期 `📅`。
- 可选：优先级、完成日期 `✅`、说明 / 备注。
- checkbox 任务行只放任务标题、优先级、循环标记和日期等结构化字段。
- 说明 / 备注写在任务下方一层缩进的 blockquote（`<Tab>> 等待接口确认`），不追加到 checkbox 行尾，不强制 `备注：` 前缀；渲染时靠附带样式弱化显示（见「注释样式」）。
- 有子任务的父任务，标题后追加进度条 `<span class="tp">…</span>`（每子任务一段、按状态上色）——见「父任务进度条」。

## 注释样式

- 注释用缩进 blockquote，与 checkbox 子任务区分：子任务是 `- [ ]`，注释是 `> `。
- Obsidian 原生给 blockquote 弱化边框；另可装技能附带的 `任务样式.css` 片段做置灰 / 倾斜 / 浅背景强化（铺设见 SKILL「记录格式」）。

## 状态标记

| 标记 | 含义 |
| --- | --- |
| `[ ]` | 未开始 / 未完成 |
| `[/]` | 进行中 |
| `[?]` | 阻塞 / 待确认 |
| `[x]` | 已完成 |

- 新建任务默认 `[ ]`。完成时改 `[x]` 并补 `✅ YYYY-MM-DD`。完成不自动归档。
- 完成子任务后，同一父任务下 checkbox 子任务全部完成时，可自动同步完成父任务，父任务完成日期取子任务中最晚的 `✅`。
- 循环 / 长期支持 / 持续进行类父任务不自动同步完成，除非用户明确要求。
- 手动完成父任务但存在未完成子任务时，先提醒并确认。

## 父任务进度条（可选，写入式）

有子任务的父任务，标题后追加一段进度条：每个子任务一个 `<i>` 色块，按子任务状态上色，纯 HTML + CSS、无插件。

```markdown
- [ ] 父任务 <span class="tp"><i class="s-done">▰</i><i class="s-doing">▰</i><i class="s-blk">▰</i><i class="s-todo">▱</i></span> 🛫 2026-06-22 📅 2026-06-30
```

- **位置**：放在标题之后、日期（`🛫`）之前。**读取标题 / 日期或生成报告时，先剥离行内 `<span class="tp">…</span>` 再匹配**，别把进度条并进标题。
- **段数与顺序**：一个子任务一个 `<i>`，按 `已完成 → 进行中 → 阻塞 → 未开始` 排序，读成从左到右的完成度。
- **状态 → 类名**：`[x]`→`s-done`、`[/]`→`s-doing`、`[?]`→`s-blk`、`[ ]`→`s-todo`。
- **字形**：已完成 / 进行中 / 阻塞用 `▰`，未开始用 `▱`——没开 CSS 片段时仍是一条黑白进度条（实心 vs 空心 ≈ 已做 vs 待做）；开了片段则上色（绿 / 黄 / 红 / 灰）。
- **只给有子任务的父任务加**；叶子任务、子任务本身不加。
- **维护（写入式的代价）**：每次改子任务状态或增删子任务后，必须重算该父任务进度条。用户手动勾子任务（不经 task-hub）时进度条会暂时失真，直到 task-hub 下次重写——这是零插件写入式的固有取舍。
- 上色靠技能附带的 `任务样式.css`（与注释样式同一片段），铺设见 SKILL「记录格式」。

## 子任务规则

- 每层缩进用 1 个 Tab，不用空格。父子任务都用 checkbox。
- checkbox 行是任务；blockquote 行是说明 / 备注。说明缩进到所属任务下方一层：顶级任务说明缩进 1 Tab，子任务说明缩进 2 Tab。
- 子任务随父任务归入同一 `## 项目名` 分区，不带项目标签。
- 调整项目 = 把任务整棵移到目标 `## 项目名` 分区，子任务随父任务一起移动。

## 归档规则

- 归档必须由用户明确触发（"归档这个任务""归档项目A已完成任务""归档上周完成任务"）。
- 归档时从 `计划中.md` 移动到同年度 `已归档.md`；恢复时移回并按用户要求设置状态。
- 父任务已完成时，归档默认移动整个任务树。
- 父任务未完成但部分子任务已完成时执行部分归档：`计划中.md` 保留父任务和未完成子任务，`已归档.md` 写入同一父任务作为上下文容器、只放已完成子任务。部分归档的父任务保留真实状态，不补 `✅`。
- 恢复父任务或部分子任务前必须确认恢复范围。

## 查找和匹配

- 先按用户给出的项目、标题、父任务、日期范围缩小候选。
- 标题不唯一时不直接修改，要求补充项目、日期或父任务。
- 查看默认读当前年度 `计划中.md`；要历史 / 已完成 / 归档 / 跨年时同时读相关年度 `已归档.md`。
- 日期范围筛选同时考虑 `🛫`、`📅`、`✅`，保留父子层级。
