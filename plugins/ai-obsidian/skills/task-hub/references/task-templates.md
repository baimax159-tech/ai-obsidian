# 任务骨架模板（照抄式，降偏差）

写操作前先读 `task-format.md` 对齐格式契约；本文件是三种高频结构的**照抄骨架**——复制整段后逐字段替换占位符，减少手写导致的格式偏差（缩进、进度条、字段顺序失配）。

所有骨架都从 `## {项目名}` 分区起手：项目归属靠分区，任务行**不带**项目标签。落笔前先按 task-format「项目集合」扫描 `计划中.md` / `已归档.md` 已有的 `## 项目名` 分区——命中即归入该分区，未命中先确认是「新建分区」还是「已有项目的别名/笔误」，不臆造重复分区。

## 占位符约定

- `{...}`：必填，替换为真实值后占位符不得残留。
- `{...?}`：可选，用则替换整行、不用则**删掉整行**。
- 日期一律 `YYYY-MM-DD`；每层缩进 **1 个 Tab**，不用空格。
- 优先级取 `⏫`（最高）/ `🔺`/ `🔼`/ `🔽`（最低）之一，无则删。

---

## 骨架一：叶子任务（无子任务）

写入 `任务/{YYYY}/计划中.md` 对应 `## {项目名}` 分区。叶子任务**不加**进度条。

```markdown
## {项目名}

- [ ] {标题} {优先级?} 🛫 {开始日期} 📅 {计划完成日期}
	> {说明?}
```

---

## 骨架二：带子任务的父任务

父任务标题后紧跟进度条（`🛫` 之前）；新建时每个子任务一段、全部 `s-todo ▱`。段数 = 子任务数。

```markdown
## {项目名}

- [ ] {父任务标题} <span class="tp"><i class="s-todo">▱</i><i class="s-todo">▱</i></span> 🛫 {开始日期} 📅 {计划完成日期}
	> {父任务说明?}
	- [ ] {子任务一} 🛫 {开始日期} 📅 {计划完成日期}
	- [ ] {子任务二} 🛫 {开始日期} 📅 {计划完成日期}
		> {子任务说明?}
```

- 子任务数 ≠ 2 时，按实际数量增删 `- [ ]` 子任务行**与**进度条里的 `<i>` 段，两者一一对应。
- 之后每次改子任务状态或增删子任务，按 `已完成→进行中→阻塞→未开始` 重排、重算进度条（写入式，详见 task-format「父任务进度条」）。

---

## 骨架三：部分归档（父任务未完成，部分子任务已完成）

仅在用户明确触发归档时使用。父任务未完成、部分子任务已完成 → 一棵树拆两侧写。父任务两侧都保留**真实状态**，容器侧不补 `✅`。

### 计划中侧（`计划中.md` 保留父任务 + 未完成子任务）

进度条只保留**未完成子任务**对应的段。

```markdown
## {项目名}

- [ ] {父任务标题} <span class="tp"><i class="s-todo">▱</i></span> 🛫 {开始日期} 📅 {计划完成日期}
	- [ ] {未完成子任务} 🛫 {开始日期} 📅 {计划完成日期}
```

### 已归档侧（`已归档.md` 同一父任务作上下文容器，只放已完成子任务）

容器父任务**不加进度条**（进度维护留在计划中侧），保留真实未完成状态、不补 `✅`。

```markdown
## {项目名}

- [ ] {父任务标题} 🛫 {开始日期} 📅 {计划完成日期}
	- [x] {已完成子任务} 🛫 {开始日期} 📅 {计划完成日期} ✅ {完成日期}
```

- 父任务**已全部完成**时不走本骨架：直接整棵树从 `计划中.md` 移到 `已归档.md`（保留原进度条）。
- 恢复前先与用户确认恢复范围，再从 `已归档.md` 移回并按要求设置状态。
