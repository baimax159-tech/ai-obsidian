# 任务中枢（task-hub）

一个技能管起 Obsidian 任务的全生命周期：**创建 / 修改 / 归档 / 报告**四类操作读写同一份 Markdown 记录。记录是纯 checkbox，给人编辑、给模型读写。

## 依赖

- 全部操作（创建 / 修改 / 归档 / 报告）纯 Markdown + HTML，**无插件依赖**。
- 周报表格用纯 HTML `<table>` + `rowspan` 渲染，Obsidian 阅读视图 / GitHub 均原生支持，项目列跨行合并不依赖 AnyBlock。

## 记录格式（所有操作的共同契约）

任务库位于 `<项目根目录>/任务/{YYYY}/`，按 `## 项目名` 分区（项目靠分区归属，任务行不带项目标签），任务行形如
`- [状态] 标题 优先级 🛫 开始 📅 计划 ✅ 完成`，说明 / 备注写成任务下方一层缩进的 blockquote（`> ...`）；有子任务的父任务在标题后追加一段进度条 `<span class="tp">…</span>`（每子任务一段、按状态上色）。
完整格式契约（状态标记 `[ ] [/] [?] [x]`、优先级 `⏫🔺🔼🔽`、子任务缩进、注释 blockquote、父任务进度条、项目集合扫描、归档规则）见 **`references/task-format.md`**——**任何写操作前先读它对齐格式**。

注释与进度条样式（可选）：`assets/任务样式.css` 一并管注释弱化与父任务进度条上色。Obsidian 原生已弱化 blockquote、进度条无 CSS 时也是黑白 `▰▱` 条；要置灰注释 + 彩色进度条，用 `Read` 读 `assets/任务样式.css`、`Write` 到库根 `.obsidian/snippets/任务样式.css`（幂等：已存在则跳过），再在 Obsidian 设置 → 外观 → CSS 片段开启。

## 能力一：创建

- 以当前项目根目录为任务库根；用户显式指定路径时以其为准。新任务写入 `任务/{YYYY}/计划中.md`。
- 必填：标题、项目、`🛫` 开始、`📅` 计划完成；缺失先用 `AskUserQuestion` 确认，不猜。
- 涉及项目时先扫描现有 `## 项目名` 分区（见 task-format 的"项目集合"）：命中即归入；不在集合内先确认是新建分区还是已有项目别名，避免重复分区。
- 用户只给大目标时，主动建议拆成父任务 + 子任务；子任务不足以覆盖时继续确认。
- **落笔照抄骨架**：叶子任务用 `references/task-templates.md` 骨架一、带子任务父任务用骨架二，复制整段后逐字段替换占位符，降低缩进 / 进度条 / 字段顺序的手写偏差。

## 能力二：修改

调整已存在任务前先读 **`references/task-editing.md`**：所有修改都走它的**定位协议**（项目缩范围 → 命中数分流 → 原行回显确认）再进 A/B/C 分叉，动手前必确认锁对了行。摘要：

- **A 改状态 / 时间**：`[ ]/[/]/[?]/[x]` 间切换或改 `🛫`/`📅`/`✅`，完成补 `✅` 不自动归档；子任务全完成可同步父任务。
- **B 移项目**：整棵任务移到目标 `## 项目名` 分区，子任务随父任务一起走。
- **C 改结构**：增删 / 拆分 / 合并父子任务后重算父进度条；执行前确认影响范围。

## 能力三：归档 / 恢复

- 只在用户明确触发时移动任务文件（`计划中.md` ↔ `已归档.md`）；完成任务不等于归档。
- 父任务未完成但部分子任务完成时支持部分归档，规则见 `references/task-format.md`，两侧写法照抄 `references/task-templates.md` 骨架三。

## 能力四：报告

- 支持日 / 周 / 月 / 季度 / 半年 / 全年报告，写入 `任务/{YYYY}/` 对应报告文件，最新在头部。
- 完整格式、日期范围、状态映射、各报告模板见 **`references/reporting.md`**——生成前先读它。
- 默认直接写入报告文件，会话里只回写入路径与覆盖范围；用户明确要"在聊天里展示"时才输出正文。
- 不编造完成日期、比例、业务指标；无量化指标时不写数字。

## 交互原则

- 熟练用户优先用简短口令（"新增任务：标题""完成：标题""归档项目A已完成任务""生成本周周报"）；新用户或信息不全时给模板并追问最少必要项。
- 日期精度默认按天；用户未给时分不追问时分。
- 只管理任务与报告，不改动 `首页.md` 等非任务文件。

## 错误处理

| 场景 | 处理 |
| --- | --- |
| 缺必填项（标题 / 项目 / 开始 / 计划完成） | `AskUserQuestion` 追问，不猜测 |
| 项目名不在现有分区 | 先确认：新建分区 or 已有项目别名归一化 |
| 标题不唯一 | 要求补项目 / 日期 / 父任务，不直接改 |
