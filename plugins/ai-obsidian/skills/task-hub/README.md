# task-hub — Obsidian 任务中枢

一个技能管起 Obsidian 任务的**完整生命周期**：创建、修改、归档 / 恢复、生成报告。

## 设计：单一记录源

- **记录**：`任务/{YYYY}/计划中.md`、`已归档.md` —— 纯 Markdown checkbox，人可读可编、模型可读写。
- 创建 / 修改 / 归档 / 报告四类操作读写同一份记录，格式逐字一致。
- **注释**：任务下方缩进 blockquote（`> ...`），与 checkbox 子任务区分；可选装 `任务样式.css` 片段做置灰 / 倾斜 / 浅背景。
- **进度条**：有子任务的父任务，标题后一段彩色进度条（每子任务一段，按状态绿 / 黄 / 红 / 灰上色），纯 HTML + CSS 无插件；与注释同一 `任务样式.css` 片段上色，未装时降级为黑白 `▰▱` 条。
- 只管理任务与报告，**不改动仓库首页等非任务文件**。

## 能力

| 能力 | 说明 | 依赖 |
|------|------|------|
| 创建 | 新增任务 / 父子任务，按项目分区，必填项缺失先追问 | 无 |
| 修改 | 状态 / 时间 / 项目 / 优先级 / 结构调整 | 无 |
| 归档 / 恢复 | 显式触发在 `计划中` ↔ `已归档` 间移动，支持部分归档 | 无 |
| 报告 | 日 / 周 / 月 / 季度 / 半年 / 全年报告 | 无 |

## 使用

```
/ai-obsidian:task-hub
```

或直接口令：

```
新增任务：接口联调，7月1日开始，7月10日完成，项目 支付
完成：接口联调
归档项目 支付 已完成任务
生成本周周报
```

## 依赖插件

**无。** 全部操作（创建 / 修改 / 归档 / 报告）纯 Markdown + HTML。周报表格用纯 HTML `<table>` + `rowspan`，项目列跨行合并由 Obsidian 阅读视图 / GitHub 原生渲染，不再依赖 AnyBlock。

## 与其它 ai-obsidian 技能的关系

task-hub 借鉴了 `work-management`（任务格式契约）与 `report-generation`（报告规范）并合为一个自足技能。task-hub 在其上精简了记录格式：项目靠 `## 项目名` 分区归属（任务行不带 `#project/` 标签）、注释用缩进 blockquote。
